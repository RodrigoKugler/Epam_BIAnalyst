# BI Analyst Training – Quality Checklist
Use this checklist before advancing any module to the next phase. Authors complete the **Author Checks**; reviewers complete the **Reviewer Checks**.

## Author Checks
- [ ] Topic scope matches `interview_preparation.txt` outline and project folder.
- [ ] Overview explains purpose and relevance in 2–3 concise paragraphs.
- [ ] Key Concepts section lists 4–7 bullet points covering fundamentals and terminology.
- [ ] Hands-on Examples include at least one executable snippet or step-by-step walkthrough.
- [ ] Practice Questions include 5 items: 3 theory, 2 applied/scenario-based.
- [ ] Interview Tips highlight common pitfalls and talking points (minimum 3 bullet points).
- [ ] Further Reading cites 3–5 reputable sources with working links.
- [ ] Tone is practical, action-oriented, and aligned with BI Analyst interviews.
- [ ] Markdown uses headings starting at `##`, code fences for snippets, and tables when necessary.
- [ ] Internal references link to shared resources (glossary, templates) when relevant.
- [ ] All examples use anonymized data and reference standard tools (SQL, Power BI, etc.).
- [ ] Draft word count is within 800–1,100 words unless otherwise specified.

## Reviewer Checks
- [ ] Facts, definitions, and examples verified for accuracy.
- [ ] Terminology consistent with other approved modules (cross-check `shared/glossary.md`).
- [ ] Scenarios reflect realistic BI Analyst interview expectations.
- [ ] Links validated (no 404s, up-to-date resources).
- [ ] Formatting passes markdownlint (headings, spacing, list markers).
- [ ] Feedback captured in `content_reviews/<subject>.md` with severity tags.
- [ ] All checklist items satisfied; outstanding items documented with remediation plan.

## Final Gate
- [ ] Reviewer approval recorded (initials + date) in review log.
- [ ] Gate status updated in `shared/quality_plan.md` sign-off table.
